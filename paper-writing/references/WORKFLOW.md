# 论文写作详细工作流

## 6 阶段工作流程

---

## 阶段 1：选题与文献调研

### 目标
- 确定具体研究方向
- 收集相关文献
- 识别研究空白

### 任务清单

1. **文献搜索**
   - Google Scholar: 关键词搜索
   - 知网/万方: 中文文献
   - ArXiv: 最新预印本
   - 会议/期刊论文: 精选高质量文献

2. **文献分类**
   - 按主题分类（使用文件夹或文献管理工具）
   - 标注重要性等级（必读/重要/参考）
   - 记录核心观点和引用关系

3. **研究空白识别**
   - 列出现有方法列表
   - 分析每个方法的局限性
   - 找出未被充分研究的问题

### 产出
- `literature_notes.md` - 文献笔记
- `research_gap.md` - 研究空白分析
- 初步参考文献列表

---

## 阶段 2：论文结构规划

### 目标
- 制定论文大纲
- 设计章节结构
- 明确各章节内容

### 任务清单

1. **确定论文类型**
   - 研究论文 (Research Paper)
   - 综述论文 (Survey/Review)
   - 会议论文 (Conference Paper)
   - 学位论文 (Thesis/Dissertation)

2. **大纲设计**
   根据论文类型选择结构：

   **研究论文标准结构**：
   ```
   Abstract
   1. Introduction (1-2页)
   2. Related Work (2-3页)
   3. Methodology (3-4页)
   4. Experiments (3-4页)
   5. Discussion (1-2页)
   6. Conclusion (0.5-1页)
   References
   ```

   **综述论文标准结构**：
   ```
   Abstract
   1. Introduction
   2. Background
   3. Main Categories
     3.1 Category 1
     3.2 Category 2
     ...
   4. Comparison and Analysis
   5. Open Challenges
   6. Future Directions
   7. Conclusion
   References
   ```

3. **章节细化**
   - 每个章节列出 3-5 个要点
   - 标注预计篇幅
   - 列出需要的图表

### 产出
- `outline.md` - 详细大纲
- `WRITING_PLAN.md` - 写作计划

---

## 阶段 3：文献综述撰写

### 目标
- 系统梳理现有研究
- 分析现有方法优缺点
- 引出本研究的必要性

### 任务清单

1. **文献组织**
   - 按时间顺序或主题分类
   - 突出重要里程碑工作
   - 建立方法之间的联系

2. **综述结构**
   ```markdown
   ## 2. Related Work

   ### 2.1 [领域A]研究现状
   早期方法 [1,2] 主要关注...
   近年来，[3,4] 提出了...
   然而，这些方法存在...

   ### 2.2 [领域B]研究现状
   与[领域A]不同，[领域B]的研究...

   ### 2.3 研究空白
   尽管上述方法取得了进展，但仍存在...
   具体而言：(1)...; (2)...; (3)...
   ```

3. **对比分析**
   - 创建方法对比表
   - 分析各方法的适用场景
   - 指出技术演进脉络

### 产出
- `related_work.md` - 相关工作草稿
- 方法对比表

---

## 阶段 4：方法与实验设计

### 目标
- 详细描述提出的方法
- 设计合理的实验方案
- 定义评估指标

### 任务清单

1. **方法描述**
   - 总体框架图
   - 各模块详细说明
   - 算法伪代码（如适用）

2. **实验设计**
   - **数据集选择**
     - 公共数据集
     - 自建数据集
     - 数据预处理

   - **评估指标**
     - 准确率相关：Accuracy, Precision, Recall, F1
     - 效率相关：Time, Memory, FLOPs
     - 鲁棒性相关：Robustness tests

   - **对比方法**
     - 选择 SOTA 方法
     - 选择经典方法
     - 选择消融实验变体

3. **实现计划**
   - 编程语言和框架
   - 硬件环境
   - 开源代码引用

### 产出
- `method_design.md` - 方法设计文档
- `experimental_plan.md` - 实验方案

---

## 阶段 5：论文正文撰写

### 目标
- 按章节顺序撰写
- 保持学术语言规范
- 确保逻辑连贯

### 撰写顺序建议

**推荐顺序**：
1. Method（最确定的内容）
2. Experiments（有数据支撑）
3. Related Work（基于对文献的理解）
4. Introduction（最后写，能更好定位贡献）
5. Abstract（最后提炼）
6. Conclusion（总结全文）

### 写作要点

**每段结构**：
```
主题句 → 支持证据 → 分析讨论 → 过渡句
```

**语言示例**：
- ✅ "Experimental results demonstrate that..."
- ✅ "The proposed method achieves..."
- ❌ "Our method is the best..."
- ❌ "Obviously..."

### 产出
- `manuscript.md` - 完整论文草稿
- 图表文件

---

## 阶段 6：修改与投稿

### 目标
- 多轮修改完善
- 格式符合投稿要求
- 准备投稿材料

### 任务清单

1. **内容修改**
   - 自我审查（使用检查清单）
   - 请同行评审
   - 根据反馈修改

2. **语言润色**
   - 语法检查
   - 拼写检查
   - 学术用语规范化

3. **格式调整**
   - 模板选择（IEEE, ACM, Springer等）
   - 图表编号
   - 参考文献格式

4. **投稿准备**
   - Cover Letter
   - 作者信息
   - 利益冲突声明
   - 代码/数据开源链接

### 产出
- 投稿版本 PDF
- 投稿辅助材料

|------|----------|----------|
| 文献调研 | 2-4周 | 1-2周 |
| 结构规划 | 3-5天 | 1-2天 |
| 文献综述 | 1-2周 | 3-5天 |
| 方法实验 | 4-8周 | 2-4周 |
| 正文撰写 | 2-3周 | 1-2周 |
| 修改投稿 | 2-4周 | 1-2周 |
| **总计** | **3-5个月** | **2-3个月** |
