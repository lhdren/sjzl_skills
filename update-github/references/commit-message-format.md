# 提交消息格式指南

## 基本原则

1. **使用现在时态** - "添加" 而不是 "添加了"
2. **首字母小写** - "修复 bug" 而不是 "修复 Bug"
3. **以动词开头** - "更新技能" 而不是 "技能更新"
4. **简洁明了** - 标题不超过 50 个字符

## 技能更新标准格式

### 格式模板

```
更新技能: <简要描述本次更新的核心内容>

更新时间: 2026-01-30 14:30:00
变更内容:
- 新增: update-github (GitHub 自动更新工具)
- 修改: skill-creator (优化初始化脚本)
- 删除: old-skill (已废弃)

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| 更新技能 | 固定前缀，标识这是技能更新提交 | 更新技能: 新增 GitHub 自动更新 |
| 更新时间 | 本次提交的时间戳 | 2026-01-30 14:30:00 |
| 变更内容 | 详细列出所有变更 | 新增/修改/删除 的技能列表 |
| Co-Authored-By | 协作者署名（固定格式） | Claude Sonnet 4.5 <noreply@anthropic.com> |

### 变更类型

- **新增**: 新创建的技能
- **修改**: 更新了 SKILL.md 或相关文件
- **删除**: 移除的技能
- **重构**: 代码重构但功能不变

## 示例

### 示例 1: 新增单个技能

```
更新技能: 新增 GitHub 自动更新工具

更新时间: 2026-01-30 14:30:00
变更内容:
- 新增: update-github (自动化 git 操作和技能同步)

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### 示例 2: 批量更新

```
更新技能: 批量更新多个技能文档

更新时间: 2026-01-30 15:45:00
变更内容:
- 修改: pdf (新增表单填写功能)
- 修改: docx (修复批注编码问题)
- 修改: xlsx (添加公式支持)

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### 示例 3: 自定义消息

如果使用 `--message` 参数，可以完全自定义提交消息：

```bash
python update_skills.py --message "修复紧急 bug: 空指针问题"
```
